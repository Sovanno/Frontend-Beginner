The Power of Semantic HTML: Why Correct Code Matters

Published: 15 January 2025 • Author: Anna Webber • 6 minutes of reading

In the world of web development, you can often hear the phrase: write semantic HTML. But what does this actually mean and why is it so important? In this article, we will break down how proper markup affects SEO, accessibility, and long-term maintainability of your code.

What is semantic markup?

Semantic HTML is an approach to markup where HTML tags are used for their intended purpose, rather than for visual styling. Instead of using <div> for everything, you choose tags that describe the meaning of the content contained within them.

Comparison: semantic vs. non-semantic code

Let's look at a simple example of creating a button. 


    <!-- ❌ Non-semantic approach -->
            <div class="button" onclick="submit()">Send</div>
        
    <!-- ✅ Semantic approach -->
            <button type="submit">Send</button>

In the second case, the browser, screen reader, and search engine immediately understand that this is a button, not just an element with a class. This provides many advantages, which we will explore further.

Key Benefits of Semantic HTML

1. Improved SEO

Search engines, such as Google, use HTML tags to understand the structure and content of a page. Proper use of headings (<h1> — <h6>), lists, and other semantic elements helps robots index your site more accurately and improve its positions in search results.

2. Increased Availability (Accessibility)

Screen readers and other assistive technologies rely on semantic markup to correctly interpret content for people with visual impairments. For example:

  • Headings allow quick navigation through the page using a keyboard.
  • Lists inform the user how many items they can expect to see.
  • The <main> tag allows jumping directly to the main content, skipping repeated blocks.

3. Code cleanliness and maintainability

When the document structure is obvious, it is easier to read, maintain, and hand over to other developers. This reduces the likelihood of errors and speeds up the implementation of new features. As one famous developer said:

Semantic HTML is not about design. It is about meaning. When you write code, you are first and foremost communicating with other developers and with the machines that will read your code.

— Tim Berners-Lee, creator of the World Wide Web

Practical examples of semantic tags

Tag <figure> и <figcaption>

These tags are used to embed illustrative content — images, diagrams, code blocks — with a caption. They link the image with its description, which is important for accessibility and context.

Quotes and links

The tags <blockquote> (for block quotes) and <q> (for inline quotes) are used for embedding quotations. It is always important to specify the source using <cite>.

According to developer Jeremy Keith, semantic HTML is the foundation of progressive enhancement, and it is hard to disagree with that.

Code and technical documentation

For embedding code, use <code> for inline strings and <pre> for blocks. This preserves formatting and improves readability. 

<!-- Example of semantic page structure -->
        <body>
          <header>
            <nav>...</nav>
          </header>
          
          <main>
            <article>
              <h1>Article title</h1>
              <p>Text...</p>
            </article>
          </main>
          
          <footer>
            <p>Copyright</p>
          </footer>
        </body>

Conclusion

Semantic HTML is not just a good practice. It is the foundation of the modern web, which ensures accessibility, SEO optimization, and long-term maintainability of your projects. Start applying these principles today, and you will notice how your code becomes cleaner and your users — more grateful.

Want to learn more? I recommend checking out the MDN documentation on HTML and studying the WAI-ARIA specifications for in-depth work with accessibility.
